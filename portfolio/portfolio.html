프로젝트 소개

OVERVIEW

실시간 상호작용을 통한 건전한 토론 플랫폼

익명성에 기대어 발생하는 무분별한 비난이 아닌, 논리적인 근거를 바탕으로 건강하게 소통할 수 있는 공간을 지향합니다. 찬반 투표를 통해 여론의 향방을 실시간으로 확인하고, 깊이 있는 의견 교환을 위해 계층형 댓글과 실시간 채팅 기능을 제공합니다. 단순한 커뮤니티를 넘어 사용자 간의 신뢰를 구축할 수 있는 장치를 마련했습니다.

KEY FEATURES
  • 실시간 투표 시스템: 동시성 이슈를 고려한 투표 집계 로직
  • WebSocket 채팅: STOMP 프로토콜을 이용한 안정적인 메시징
  • 계층형 댓글: Self-join 구조의 고성능 대댓글 시스템
  • 모니터링 CMS: 신고 내역 관리 및 사용자 제재 시스템
  • 비동기 알림: 토론 시작 및 댓글 등록 시 실시간 푸시

기술 스택

BACKEND
  • Spring Boot 3.2: 생산성 높은 애플리케이션 구축
  • Spring Security + JWT: 무상태성 인증 시스템 구축
  • Spring Data JPA: 객체 지향적 데이터 관리 및 쿼리 최적화

  • RESTful API: 규격화된 인터페이스 설계 및 문서화
FRONTEND
  • React 18: 컴포넌트 기반 UI 개발 및 가상 DOM 활용
  • Vite: 빠른 빌드 도구 도입으로 개발 생산성 향상
  • Context API: 효율적인 전역 상태 관리 및 리팩토링 용이성

  • SockJS: 브라우저 호환성을 고려한 WebSocket 대체 기술
  • Styled Components: 유지보수가 용이한 CSS-in-JS 도입
INFRASTRUCTURE
  • AWS EC2: 클라우드 가상 서버 환경 구축
  • Nginx: 리버스 프록시 및 로드 밸런싱 구성
  • MySQL 8.0: 관계형 데이터베이스를 통한 데이터 정규화
  • Let's Encrypt: 보안 통신을 위한 SSL/TLS 적용
  • Git/GitHub: 버전 관리 및 팀 협업 워크플로우 운영
  • Ubuntu 22.04: 리눅스 서버 운영 체제 관리

문제 해결 (1)

PROBLEM

실시간 채팅 연결 실패

실제 서비스 환경에서 채팅 연결이 갑자기 끊기는 현상 발생. 서버 설정 문제로 초기 연결 단계에서 오류 확인.

SOLUTION
  • 서버에 웹소켓 전용 연결 허용 설정 추가
  • 최신 통신 규약(HTTP 1.1)이 원활하게 작동하도록 보정
  • 연결을 유지해주는 'Upgrade' 헤더 정보 누락 방지
  • 연결이 끊기지 않도록 대기 시간 설정값 최적화
PROBLEM

보안 정책에 따른 접근 차단

주소가 다른 서버끼리 데이터를 주고받을 때 웹 브라우저의 보안 정책에 의해 요청이 거부됨.

SOLUTION
  • 서버 보안 설정에 허용할 주소를 명확히 등록
  • 로그인 인증 정보를 안전하게 주고받을 수 있도록 허용
  • 실제 데이터를 보내기 전, 보안상 안전한지 미리 확인하는 과정 처리
  • 프론트엔드와 백엔드 간의 데이터 교환 규칙 통일
Nginx 설정 Spring Security WebSocket CORS Troubleshooting Proxy 설정

문제 해결 (2)

PROBLEM

자동 로그아웃 현상

로그인 유효 시간이 끝났을 때 알림 없이 끊겨 사용자가 작성 중이던 데이터를 잃어버리는 문제.

SOLUTION
  • 통신 상태를 자동으로 감지하여 로그인 만료 여부 확인
  • 로그인이 끊기기 전 미리 사용자에게 안내 메시지 표시
  • 유효 시간이 끝났을 때 매끄럽게 로그인 화면으로 이동 처리
PROBLEM

댓글 조회 시 속도 저하

대댓글이 많아지면 서버가 정보를 가져오는 횟수가 기하급수적으로 늘어나 화면 로딩이 매우 느려짐.

SOLUTION
  • 연관된 데이터를 여러 번 나누지 않고 한 번에 가져오도록 최적화
  • 화면에 필요 없는 정보는 빼고 필요한 데이터만 골라서 조회
  • 조회 성능을 개선하여 응답 속도를 약 80% 향상
